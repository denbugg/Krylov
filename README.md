# EGE Author Position Bot v1

Telegram-бот для первой версии тренажёра сочинения ЕГЭ по русскому языку.

Функционал v1:

Управление сделано через кнопки и панели. Команды оставлены только как запасной способ.

1. новая проблема;
2. отработка конкретного варианта;
3. отправка текста задания сообщением или скриншотом;
4. проверка авторской позиции через Gemini API;
5. статистика по вариантам и темам;
6. сортировка слабых вариантов по баллам.

Проверки литературного аргумента в этой версии нет. Она намеренно вынесена в следующие версии.

---

## 1. Структура данных

Бот использует два источника:

```text
data/positions.json
out/report.json
out/screens/
out/texts/
```

`data/positions.json` содержит:

```json
{
  "variants": [
    {
      "variant": 1,
      "problem": "...",
      "author_position": "...",
      "topic": "..."
    }
  ]
}
```

`out/report.json` содержит соответствие варианта файлам текста и скриншота:

```json
{
  "ok": [
    {
      "number": "1",
      "title": "...",
      "text_file": "01_....txt",
      "screenshot": "01_....png"
    }
  ]
}
```

Рядом должны лежать:

```text
out/texts/01_....txt
out/screens/01_....png
```

В архиве уже есть `data/positions.json` и `out/report.json`. Папки `out/screens` и `out/texts` созданы пустыми: скопируй туда файлы из своей локальной папки `out`.

## 2. Установка

```bash
python -m venv .venv
```

Windows PowerShell:

```powershell
.venv\Scripts\Activate.ps1
```

Linux/macOS:

```bash
source .venv/bin/activate
```

Далее:

```bash
pip install -r requirements.txt
copy .env.example .env
```

На Linux/macOS вместо `copy`:

```bash
cp .env.example .env
```

В `.env` заполни:

```text
BOT_TOKEN=...
GEMINI_API_KEY=...
```

---

## 3. Проверка данных

```bash
python scripts/validate_data.py
```

Скрипт проверит:

- есть ли 50 вариантов;
- совпадают ли номера в `positions.json` и `report.json`;
- есть ли файлы в `out/texts`;
- есть ли файлы в `out/screens`.

Если текстов/скринов нет, бот всё равно запустится, но кнопки просмотра будут писать, что файл не найден.

## 5. Управление в боте

Основное управление — кнопками.

Нижняя панель:

```text
🆕 Новая проблема
🎯 Конкретный вариант
📊 Статистика
📉 Баллы по вариантам
❌ Отмена
```

Внутренние панели:

```text
📖 Читать текст
🖼 Смотреть скрин
✍️ Писать авторскую позицию
```

Для выбора конкретного варианта используется кнопочная сетка 1–50 с перелистыванием страниц.

Команды оставлены как запасной вариант:

```text
/start
/new
/variant 25
/variants
/stats
/cancel
```

---

## 6. Оценка авторской позиции

Максимум: `3`.

```text
0 — мимо: неверная позиция, подмена проблемы, противоречие эталону.
1 — частично: зацеплена одна подтема, но мысль общая или уходит в сторону.
2 — в целом верно: смысл близкий, но есть заметная неполнота/неточность.
3 — полное смысловое попадание.
```

Модель возвращает вещественный балл `0.0–3.0`. В статистике используется последний результат по каждому варианту.

---

## 7. Как бот выбирает новую проблему

Алгоритм `/new`:

1. сначала берёт вариант, который пользователь ещё не делал;
2. по возможности выбирает тему, отличную от темы последней попытки;
3. если всё уже делалось, берёт вариант с самым низким последним баллом;
4. если и это невозможно, выбирает первый вариант из базы.

---

## 8. Что НЕ входит в v1

В v1 нет проверки:

- собственного аргумента;
- литературных фактов;
- цитат;
- фактических ошибок по произведениям;
- тонких интерпретаций героев.

Это отдельный модуль следующих версий. В будущих версиях нужно добавить:

1. базу школьных произведений;
2. паспорта героев и эпизодов;
3. проверку фактических ошибок;
4. рекомендации по безопасному эпизоду;
5. запрет на вымышленные цитаты;
6. уточнение, читал ли ученик произведение полностью или опирается на краткое содержание.

---

## 9. Логика проекта по версиям

### v1

Авторская позиция + статистика по темам.

### v2

Проверка собственной позиции и аргумента без глубокой фактологии.

### v3

Локальная база школьного канона: произведения, герои, эпизоды, безопасные трактовки.

### v4

Карта тем и аргументов ученика.

### v5+

Гуманитарный навигатор: культурный опрос, читательский дневник, рекомендации, поиск людей с близкими интересами.
